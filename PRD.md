## 1. 产品概述

### 1.1 产品名称
**Random Trader vs Index**（随机交易员大战大盘）

### 1.2 目标

* 用股票数据复现一个“随机策略”交易过程
* 以**强可视化+动画**方式展示：

  * 每周发生了什么交易（买/卖/持仓变化）
  * 最终收益如何（策略净值 vs 基准净值）
  * 是否跑赢大盘 / 跑输大盘
* 支持录屏做视频：从“数据导入 → 参数设置 → 一键播放 → 结果总结”

### 1.3 受众

* 量化课同学 / 老师 / B站观众（偏科普、好玩、可解释）

---

## 2. 核心卖点（视频效果优先）

1. **K线图作为主舞台**

* 背景是大盘（或某股票）蜡烛图
* 上面叠加两条“会动”的折线（策略净值 vs 基准净值）

2. **交易动作有“戏剧化动画”**

* 每周出现一个“交易卡片”（Week N：随机买入/随机卖出/无操作）
* K线上出现买卖标记（↑ 买 / ↓ 卖），带轻微弹跳和高亮
* 资产曲线在那一周“被拉动”变化（缓动曲线过渡）

3. **结局仪式感**

* 最后出现结算面板：总收益、年化、最大回撤、波动率、胜率等
* “跑赢/跑输大盘”用大字 + 动效（如烟花/碎纸/警报灯）
* 可导出一张“成绩单 PNG”

---

## 3. 使用流程（用户旅程）

### 3.1 快速模式（默认给录屏用）

1. 上传数据（CSV/Excel）
2. 选择基准（指数字段/某列，或默认收盘价作为基准）
3. 选择策略参数（每周、随机买卖规则、仓位规则）
4. 点击 **“开始播放”**
5. 观看：周度推进动画 + 实时指标更新
6. 点击 **“生成总结卡片 / 导出”**

### 3.2 高级模式（可选）

* 批量跑 100/1000 次随机策略，展示分布：

  * 你的这一次在分布中的位置
  * 跑赢大盘的概率（在本数据上）

## 4. 功能需求

### 4.1 数据导入与校验

**输入格式**（最少需要）：

* 日期列：Date
* OHLC：Open/High/Low/Close（用于蜡烛图）
* 可选：Volume、IndexClose（若大盘指数另有列）

**校验**：

* 日期单调递增、缺失处理（跳过/前向填充/报错）
* 周频规则：以自然周（周一开盘到周五收盘）或按数据中每 5 个交易日聚合
* 输出规范化后的 DataFrame：`daily_bars`、`weekly_bars`

### 4.2 策略引擎（随机买卖）

定义：每周一次决策（周末/周一开盘前），动作从集合中随机选择。

**策略参数（前端可调）**：

* 决策频率：每周（MVP）
* 动作集合：

  * Buy（买入）
  * Sell（卖出）
  * Hold（不动）
* 动作概率（默认均匀，也可自定义如 40/40/20）
* 仓位规则：

  * 全仓/半仓/固定金额
  * 是否允许做空（默认不允许）
* 交易成本：

  * 手续费（bps）
  * 滑点（bps）
* 初始资金：如 100,000
* 是否分红复权（取决于数据，有就做）

**输出**：

* 周度交易序列：[{week, action, price, shares, cash, position_value, total_equity}]
* 日度净值序列（用于平滑动画）：equity_curve_daily
* 基准净值：benchmark_curve_daily（买入持有或指数净值）

### 4.3 结果指标

* 总收益率、年化收益率
* 年化波动率
* 最大回撤（MDD）
* 夏普（可选）
* 跑赢大盘的差值：Alpha（这里可简单定义为策略总收益 - 基准总收益）

### 4.4 前端可视化与动画（重点）

**主舞台：蜡烛图 + 双净值线**

* 蜡烛图（OHLC）
* 叠加两条线：

  * 策略净值线（Equity）
  * 基准净值线（Benchmark）
* 两条线随时间推进“逐点生长”（像画线一样）

**每周事件动画**

* 时间轴推进到下一周：

  * 当前周蜡烛区域高亮（背景淡黄闪一下）
  * 出现“交易卡片”（右侧弹出）：

    * Week 12 | Action: BUY | Price: 3210 | Shares: 12.3
  * 图上落下买卖标记（带弹跳）
  * 上方 KPI 面板数字滚动更新（odometer 效果）

**播放控制**

* 播放 / 暂停 / 下一周 / 上一周
* 播放速度：0.5x / 1x / 2x / 4x
* 进度条拖动跳转（跳转后动画直接到该状态）

**结算面板**

* 动画结束后覆盖层：

  * “你跑赢了吗？”大标题
  * 指标卡片
  * 导出按钮：PNG / JSON（交易记录）/ CSV（净值）

### 4.5 批量模拟（可选增强）

* 输入：模拟次数 N（100/1000）
* 输出：分布图（你的收益在分布中哪个分位）
* 动画：快速闪回“每次结果点”最终形成散点/直方图

## 6. 关键交互细节（给前端/UI）

### 6.1 KPI 面板（顶部）

* 当前日期
* 当前总资产（滚动数字）

* 出现时：从右侧滑入 + 轻微缩放
* 颜色区分动作：买入/卖出/持有
* 卡片可点击：跳转图表到该周位置

### 6.3 图表动画建议

* 蜡烛图固定渲染（全量一次画完）
* 线条用“mask/clip”实现逐步增长（更丝滑）
* buy/sell 标记出现带“弹跳”+“阴影”
* 当前周区域半透明高亮矩形跟随

## 7. 技术方案（后端 Python + 前端 HTML）

### 7.1 后端（Python）

建议：FastAPI

**模块拆分**

* `data_loader`: 读取 CSV/Excel、列映射、清洗
* `resampler`: 日频 → 周频聚合
* `strategy_random_weekly`: 随机策略执行
* `metrics`: 指标计算（MDD、年化等）
* `exporter`: 输出 JSON/CSV/PNG（PNG 可前端生成更方便）

**API（MVP）**

* `POST /api/upload`：上传文件 → 返回 dataset_id + 列信息 + 预览
* `POST /api/run`：传入 dataset_id + 参数 → 返回

  * daily candles（用于画图）
  * daily equity curve
  * daily benchmark curve
  * weekly trades list（用于周度动画事件）
  * metrics summary
* `POST /api/simulate`（可选）：N 次模拟 → 返回分布统计

### 7.2 前端（HTML）

建议：

* UI：原生 + Tailwind / 或 React（但你说“前端使用 html”，可以理解为 Web 前端即可）
* 图表：**TradingView Lightweight Charts** 或 **Plotly**

  * Lightweight Charts：蜡烛图和性能都很好，适合丝滑动画
  * Plotly：开发快，但动画性能可能略吃力

**动画**

* GSAP / Framer Motion（React）
* 或纯 CSS + requestAnimationFrame 控制线条进度

## 8. 数据结构（前后端契约）

```json
[
  {"t":"2024-01-02","o":10.1,"h":10.8,"l":9.9,"c":10.5,"v":123456},
  ...
```

### 8.2 Equity curves（日频）

```json
{
  "equity":[{"t":"2024-01-02","v":1.000},{"t":"2024-01-03","v":1.012}],
  "benchmark":[{"t":"2024-01-02","v":1.000},{"t":"2024-01-03","v":1.006}]
```

```json
[
  {"week":"2024-W01","t":"2024-01-05","action":"BUY","price":10.5,"shares":500,"cash":47500,"equity":100000},
  ...
]
```

## 9. 异常情况

* 数据缺列 / 日期无法解析 → 明确提示“需要 Date + OHLC”
* 数据存在空洞（缺交易日） → 提供三种处理策略（跳过/填充/报错）
* 周聚合后不足 2 周 → 不允许运行（提示样本太短）
* 全程随机都 Hold → 允许，但总结显示“无交易”
* 手续费导致资金不足无法买入 → 动作自动降级为 Hold，并在卡片里标注“资金不足”

## 10. 验收标准（MVP 必须达成）

1. 课堂数据（CSV/Excel）能成功渲染蜡烛图
2. 点击运行后，能生成：

   * 策略净值线 + 基准净值线
   * 周度交易事件列表
   * 指标汇总
3. 播放动画：

   * 每周推进时有高亮、卡片弹出、买卖标记出现
   * KPI 数字跟随变化（至少总资产与收益%）
4. 结算页能显示跑赢/跑输，并能导出交易记录 JSON
